03 / Defensive Measures

MITIGATION
STRATEGIES

Layered defences across prevention, detection, and recovery — applied at the OS, memory, network, and process levels.

01
STACK CANARIES & BOUNDS CHECK

Sentinel values placed between local variables and return address detect overflow before control flow is hijacked. Combined with compiler-enforced bounds checking on array operations.

PREVENTIONMEMORYBUFFER OVERFLOW
02
ASLR + DEP / NX BITS

Address space layout randomisation defeats hardcoded pointer exploits. Non-executable stack and heap pages (DEP/NX) prevent shellcode execution even after successful overflow.

PREVENTIONMEMORYROP
03
CODE SIGNING & BINARY INTEGRITY

Cryptographic signatures on all binaries and kernel modules detect trapdoor insertion and supply-chain compromise at load time. Verified boot chains ensure integrity from firmware up.

DETECTIONOS-LEVELTRAPDOOR
04
DNSSEC + CACHE HARDENING

DNSSEC signed records prevent forged DNS responses. Source-port randomisation and TTL anomaly monitoring close the transaction-ID prediction attack surface against resolvers.

PREVENTIONNETWORKCACHE POISON
05
ATOMIC OPS & MUTEX LOCKS

Kernel-level atomic check-and-set operations eliminate TOCTOU race windows. Mandatory access control policies restrict which processes can access sensitive resources, shrinking the attack surface.

PREVENTIONCONCURRENCYTOCTOU
06
SNAPSHOT ROLLBACK & ISOLATION

VM and container snapshots enable sub-minute recovery to a known-good state. Process sandboxing via seccomp and namespaces limits blast radius — a compromised process cannot escalate.
